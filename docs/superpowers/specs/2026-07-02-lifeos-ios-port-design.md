# LifeOS iOS Port Design

## Goal

Build a standalone native iOS version of LifeOS in `ios/` that mirrors the Android app's offline-first behavior, primary modules, seed data, visual language, and release workflow closely enough for ad hoc `.ipa` testing on a registered iPhone.

## Distribution Decision

Use direct ad hoc `.ipa` distribution instead of TestFlight. This requires a paid Apple Developer account, an iOS Distribution certificate, an explicit App ID, the tester iPhone UDID registered in the Apple Developer portal, and an ad hoc provisioning profile that includes that UDID. The Windows workspace can create and verify source, docs, and packaging scripts, but the signed `.ipa` must be produced on macOS with Xcode.

## Architecture

The iOS app is a native SwiftUI app generated by XcodeGen from `ios/LifeOS/project.yml`. It uses no third-party runtime dependencies and no network APIs. Local data lives in an Application Support JSON store using `Codable`, which keeps the first port straightforward to audit, back up, export, and sign. Core business behavior is separated from SwiftUI into model, logic, persistence, export, theme, and notification files so XCTest can validate the Android-equivalent rules.

## Feature Scope

The first iOS version implements the user-visible LifeOS modules:

- Dashboard command center with greeting, today's progress, command search, quick capture, smart daily plan, review summaries, trends, focus timer, reminders, and module navigation.
- Habits with default seeded habits, daily completion, streaks, add/edit/delete behavior.
- Tasks with folders, due dates, priority, status, tags, search/filter behavior, and quick-captured recurrence expansion.
- Calendar with app-only events and selected-day context.
- Money with seeded categories, expense logging, monthly total, category breakdown, edit/delete.
- Fitness with seeded muscle groups/exercises, set logging, workout history, and volume.
- Level Up with journal, mood tracking, mood trend, reading/watch list.
- Notes with folders/tags-ready local note records, pinning, daily notes, search, and soft delete.
- Settings with theme selection, GBP default currency, privacy/widget toggles, notification permission, local export, and layout/nav customization state.

The iOS version intentionally preserves the Android app's local-only invariant: no accounts, no cloud sync, no analytics, no remote config, no downloadable assets, and no network calls.

## UI Design

SwiftUI recreates the Android app's structure rather than copying Material components directly. The shell uses a themed top bar, scrollable content area, rounded LifeOS cards, chips, progress strips, and a floating bottom navigation bar. Theme tokens cover Light, Dark, Midnight, Forest, Sunrise, Samurai, 8-bit Console, Sakura, Adaptive Glass, Dark Neumorphism, Glassmorphism, Rainy Focus, Lofi Girl, Anime Cyberpunk, and the newer Android theme enum entries with iOS-safe color treatments.

## Data Flow

`LifeOSStore` owns the app state as an observable object. Screens call store commands to mutate data. Store commands validate input, update state, persist immediately to JSON, and schedule/cancel iOS local notifications where applicable. Derived dashboard, review, search, trend, streak, and quick-capture behavior lives in pure helpers under `LifeOSLogic.swift`.

## Error Handling

User input errors are handled inline by ignoring blank submissions, clamping invalid numeric values, and keeping destructive actions local to the selected record. Persistence failures are exposed through `lastError` so Settings can show an actionable message. Export creates a timestamped local folder and reports the exported URL for sharing through iOS.

## Testing

XCTest targets cover the highest-risk parity logic: habit completion math, greetings, streaks, command search ordering, quick capture parsing and recurrence expansion, focus timer formatting/progress, dashboard summary construction, and store mutations. Because this workspace has no Xcode or Swift toolchain, local verification also includes a static source validator that checks required files, offline source constraints, route/theme coverage, and release packaging files.

## Release

The `release/` folder documents direct `.ipa` install paths and includes an ad hoc export options template plus OTA manifest template. The `scripts/build-adhoc-ipa.sh` script generates the Xcode project, archives the app, and exports a signed `.ipa` on macOS when the developer supplies signing values. A GitHub Actions workflow is included for future signed `.ipa` releases once Apple signing secrets are configured.

## Self-Review

- No placeholder requirements remain in this spec.
- The distribution choice matches the user's direct `.ipa` request.
- The implementation is scoped to an iOS port in `ios/`, avoiding unrelated Android worktree edits.
- The known platform blocker is explicit: signed `.ipa` creation requires macOS/Xcode and Apple signing assets.
